## 🚀 快速启动

### 1. 后端服务（已启动✅）

```bash
cd backend
pip install -r requirements.txt
python main.py
```

**访问地址：**
- API服务: http://localhost:8000
- API文档: http://localhost:8000/docs
- 测试页面: 双击 `test_api.html`

### 2. Web前端（需要Node.js）

```bash
cd frontend
npm install
npm run dev
```

访问: http://localhost:3000

### 3. 移动App（需要Node.js）

```bash
cd mobile-app
npm install
npm start
```

使用Expo Go扫码在手机上运行

## 🎯 核心功能

### 1️⃣ 个性化动态建模引擎

**代谢适应性模型**
- 使用梯度提升树预测基础代谢率
- Harris-Benedict公式作为基准
- 实时追踪代谢变化

**能量平衡预测**
- 预测未来7天体重变化趋势
- 综合摄入、运动和代谢数据
- 动态调整减重计划

**平台期识别**
- 自动检测减重平台期
- 提前预警并提供建议
- 帮助突破瓶颈

### 2️⃣ 情境感知的饮食AI顾问

**智能饮食记录**
- 📸 拍照识别食物（图像识别）
- 🎤 语音/文字输入
- 📊 自动计算营养成分

**智能食谱生成**
- 基于用户偏好的低卡替代方案
- GPT-4驱动的食谱创作
- 考虑口味和营养平衡

**情绪与渴望管理**
- 记录进食情绪
- 分析情绪性进食模式
- 触发时刻推送干预建议

**社交情境适应**
- 聚餐场景餐厅选择
- 餐前餐后饮食调整
- 保持社交与健康平衡

### 3️⃣ 自适应运动AI教练

**个性化运动处方**
- 根据体能水平定制计划
- 考虑时间和器材限制
- 周期性动态调整

**依从性优化**
- 监测运动完成情况
- 分析未完成原因
- 提供最小可行方案（5分钟运动）

**智能调整**
- 根据反馈调整强度
- 适应用户进步
- 避免过度训练

## 📊 API接口

### 用户管理
- POST `/api/users/register` - 用户注册
- GET `/api/users/{user_id}` - 获取用户信息

### 代谢建模
- POST `/api/metabolism/predict-bmr` - 预测基础代谢率
- POST `/api/metabolism/detect-plateau` - 检测平台期
- POST `/api/metabolism/predict-weight` - 预测体重变化

### 饮食管理
- POST `/api/diet/recognize-food` - 识别食物
- POST `/api/diet/log-food` - 记录饮食
- POST `/api/diet/generate-recipe` - 生成食谱
- POST `/api/diet/log-emotion` - 记录情绪

### 运动管理
- POST `/api/exercise/generate-plan` - 生成运动计划
- POST `/api/exercise/log-exercise` - 记录运动
- GET `/api/exercise/analyze-adherence/{user_id}` - 分析依从性

### 数据分析
- GET `/api/analytics/dashboard/{user_id}` - 获取仪表盘
- GET `/api/analytics/progress/{user_id}` - 获取进度报告
